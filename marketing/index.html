Why VendIQ

Vendor data shouldn't live in fifteen spreadsheets.

Procurement, legal, finance, and risk each maintain their own lists. Renewals slip. Critical-vendor exposure goes unnoticed. Spend leaks through shadow contracts. VendIQ makes the portfolio visible — and answerable — in one place.

The Problem

  • • Contract dates buried in shared drives
  • • Risk assessments isolated in OneTrust + ServiceNow
  • • Spend roll-ups stitched together quarterly
  • • "Who owns this vendor?" is nobody's job

The Solution

  • • Single Dataverse model spanning vendors, suppliers, contracts, spend, risk
  • • Expiration Radar surfaces 0–30 / 31–60 / 61–90 day windows
  • • Vendor 360° pulls all signals into one view
  • • Embedded Copilot answers questions in plain English

The Outcome

  • • Zero surprise renewals on critical vendors
  • • Risk + spend + criticality cross-referenced in one click
  • • Procurement, legal, and risk on the same page — literally
  • • Decisions in seconds, not days

Layer 1 · Presentation

Components only render

Domain components in src/components/vendiq/ + Radix/Tailwind primitives in src/components/ui/. No data fetching here.

Layer 2 · Orchestration

Hooks compose state

TanStack Query hooks in src/hooks/vendiq/ compute KPIs, filters, and cross-entity joins on top of provider data.

Layer 3 · Data Access

Adapters wrap generated code

Provider contracts in src/services/vendiq/ have mock + Dataverse implementations. src/generated/ is read-only.

Data Lifecycle

Schema-as-code from day one.

A single planning payload drives provisioning, code-generation, mock data, and the Fabric handoff. The same JSON file is the contract every team builds against.

01

Plan

Edit dataverse/planning-payload.json — the canonical schema.

02

Validate

npm run validate:schema-plan guards against drift before any write.

03

Provision

npm run schema:provision creates tables, columns, option sets, relationships.

04

Register

npm run register:dataverse wires tables into the Code App and regenerates SDKs.

05

Hand off

npm run fabric:map emits a source→target workbook for the Fabric team.
